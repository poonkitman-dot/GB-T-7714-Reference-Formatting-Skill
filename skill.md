# GB/T 7714-2015 参考文献著录 Skill（工程化改进版）

## 核心设计思想（必须遵循）
本 skill 不只是“格式模板”，而是一个 **决策驱动的结构化生成器**：

输入 → 类型判定 → 字段提取 → 标准化 → 格式输出

---

## 一、工作流程（强化版）

1. **识别是否为电子资源**（是否包含 URL / DOI / 在线访问信息）
2. **执行文献类型判定决策树（关键）**
3. **提取著录字段**（作者、题名、时间等）
4. **字段标准化处理**（日期、作者格式等）
5. **按规范拼接输出**
6. **检查缺失字段并提示**

---

## 二、文献类型判定决策树（核心改进）

### Step 1：是否为电子资源？
- 有 URL / DOI → 是 → 进入电子资源分支
- 否 → 按传统文献类型判断（M/J/D 等）

### Step 2：电子资源细分（优先级判断）

1. 是否为传统出版物的网络版本？
   - 期刊文章 → J/OL
   - 图书 → M/OL

2. 是否为机构发布的网页/公告/指南/政策文件？
   - 是 → **EB/OL（默认优先）**

3. 是否具有明确“研究报告”属性（如报告编号、研究机构发布）？
   - 是 → R/OL

4. 无法判断 → 默认 EB/OL

---

## 三、字段标准化规则（强制执行）

### 1. 日期格式（必须转换）
- 输入：2025.08.28 / 2025年8月28日
- 输出：2025-08-28

### 2. 引用日期
- 必须使用格式：[YYYY-MM-DD]

### 3. 作者处理
- 机构作为作者：直接作为“主要责任者”
- 删除“来源：”“发布单位：”等自然语言

### 4. URL
- 必须保留原始链接
- 不加“来源”或说明性文字

---

## 四、文献类型标识

| 类型 | 标识 |
|------|------|
| 专著 | M |
| 期刊 | J |
| 学位论文 | D |
| 报告 | R |
| 标准 | S |
| 专利 | P |
| 报纸 | N |
| 电子公告 | EB |

载体标识：
- OL（在线资源）

组合示例：
- EB/OL（网页）
- J/OL（在线期刊）

---

## 五、关键类型格式（精简但强约束）

### 5.1 电子资源（重点）

```
主要责任者. 题名[EB/OL]. (发布日期)[引用日期]. URL.
```

说明：
- 无需强制“出版地：出版者”
- 若存在更新时间，可放入 () 中

---

### 5.2 期刊论文

```
作者. 题名[J]. 期刊名，年，卷(期)：页码.
```

---

### 5.3 专著

```
作者. 书名[M]. 出版地：出版社，年份.
```

---

### 5.4 报告

```
作者. 标题[R]. 出版地：出版者，年份.
```

---

## 六、标点规则（严格）

- 作者后：.
- 题名后：[类型]
- 日期使用 () 和 [] 区分
- URL 结尾必须加 .

---

## 七、异常与缺失处理

### 缺失作者
- 使用题名开头

### 缺失日期
- 提示用户补充
- 若无法补充，可省略发布日期，仅保留引用日期

### 多作者
- 超过 3 人 → 前 3 人 + “等”

---

## 八、输出策略（重要）

### 单条文献
直接输出标准格式

### 多条文献
```
[1] ...
[2] ...
```

---

## 九、示例（验证用）

输入：
- 标题：高质量数据集建设指引
- 作者：国家数据局
- 日期：2025.08.28
- 引用日期：2026年4月13日
- URL：xxx

输出：
```
国家数据局. 高质量数据集建设指引[EB/OL]. (2025-08-28)[2026-04-13]. URL.
```

## 十、设计总结（供模型理解）

本 skill 的关键不在“字段拼接”，而在：

1. 类型判定优先级（特别是 EB/OL vs R/OL）
2. 日期强制标准化
3. 去自然语言噪音（如“来源”）
4. 输出最简但合规结构

目标：稳定生成、避免歧义、优先正确。

---

## 十一、不确定性处理机制（新增）

### 1. 触发条件
当出现以下情况时，必须触发不确定性机制：

- 文献类型存在歧义（如 EB/OL vs R/OL）
- 作者与出版者无法区分
- 日期字段不明确（发布日期 vs 更新日期）
- 信息缺失影响类型判断

---

### 2. 输出策略（两阶段）

#### （1）给出“最可能正确”的结果（必须输出）

```
标准格式参考文献
```

#### （2）附加不确定性提示（简洁）

- 标注：类型判断依据
- 提供备选类型（如适用）

示例：
```
国家数据局. 高质量数据集建设指引[EB/OL]. (2025-08-28)[2026-04-13]. URL.

说明：该文献为政府网站发布的在线指南，按规范优先判定为 EB/OL；若视为研究报告，也可著录为 R/OL。
```

---

### 3. 置信度标记（可选增强）

可在说明中加入：

- 高置信度：结构清晰，无歧义
- 中置信度：存在轻微类型判断
- 低置信度：关键信息缺失

示例：
```
置信度：中（EB/OL 与 R/OL 存在类型歧义）
```

---

## 十二、双标准支持机制（新增）

### 1. 触发规则

当用户出现以下表达时切换标准：

- “APA格式” → APA
- “Chicago格式” → Chicago
- “GB/T 7714” → 强制使用本标准
- 未指定 → 默认 GB/T 7714

---

### 2. 输出模式

#### 单标准模式
仅输出目标格式

#### 对比模式（推荐）
当用户未明确指定但存在跨学科需求时：

```
【GB/T 7714】
...

【APA】
...

【Chicago】
...
```

---

### 3. 核心差异自动转换规则

#### （1）作者格式
- GB/T：中文全名 / 英文大写姓
- APA：姓, 名首字母
- Chicago：姓, 名（完整）

#### （2）日期位置
- GB/T：标题后
- APA：作者后括号
- Chicago：接近出版信息

#### （3）标题大小写
- GB/T：保持原样
- APA：句式大小写
- Chicago：标题式大小写

---

### 4. 示例（跨标准）

输入：国家数据局 + 网页

输出：

```
【GB/T 7714】
国家数据局. 高质量数据集建设指引[EB/OL]. (2025-08-28)[2026-04-13]. URL.

【APA】
国家数据局. (2025, August 28). 高质量数据集建设指引. URL

【Chicago】
国家数据局. "高质量数据集建设指引." August 28, 2025. URL.
```
